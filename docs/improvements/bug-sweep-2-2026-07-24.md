## Fixed in this PR

### 1. Boss death insight coached the WORST possible line (content, ┬зEthics)
`lib/data/insights.dart` тАФ the boss bucket said *"The Tyrant blocks on turn
2; hold your damage and strike turn 3."* Sim-verified (block absorbs the
FOLLOWING player turn's attacks тАФ see the new `sim_test.dart` anchor test):

- player turn 2 (block intent shown): the Tyrant has **0 block** тАФ attacks
  land in full;
- player turn 3: attacks sink into the banked **28 block**; turn 4 into the
  attack+block intent's 20.

So the tip told the player to skip their best window and swing into the
guard тАФ a death-screen line that makes the player *worse*. Reworded
number-free and direction-correct: *"The Tyrant shields through the middle
of its cycle тАФ pour damage into the early beats, never into a raised
guard."* Regression anchor added: `enemy block from a block intent absorbs
NEXT turn's attacks`.

### 2. Burn-kill death dissolve cut off by the phase switch (UI choreography)
`lib/ui/screens/combat_screen.dart` тАФ `_endTurn`'s worst path when the burn
tick kills the enemy after an attack intent was squash 90 + contact 250 +
hit-stop 80 + knock 140 + flash tail 120 + burn beat 350 + dissolve 700 тЙИ
**1730 ms**, past the 1450 ms `terminalHold` тАФ the reward screen swapped in
while the enemy was mid-dissolve (PR #11's audit only measured the
player-death path, ~1360 ms). Fix: skip the 350 ms burn beat when the tick
kills тАФ the death choreography is the payoff тАФ bringing the worst path to
~1380 ms, inside budget. The prior sweep doc carries a correction note.

### 3. Boon screen played the title theme mid-run (audio)
`lib/audio/audio_service.dart` тАФ `musicKeyForPhase` had no case for `boon`,
so it fell through to `title_menu`. "Delve again" after a death played:
defeat sting тЖТ title theme over the boon pick тЖТ map music seconds later.
The map background already shows behind the boon screen
(`Art.backgroundForPhase`), so `boon` now maps to the `map` track like the
other run phases.

### 4. Audio-settings save wasn't atomic or serialized (robustness)
`lib/audio/settings.dart` тАФ `SettingsStore.save` was a bare
`writeAsString`, unlike MetaStore/run autosave which both document the
queue + temp-file + rename pattern as required. A slider release plus a
mute tap fire back-to-back saves that could interleave, and a crash
mid-write left truncated JSON (silently resetting the player's audio
settings on load). Now uses the same queued atomic pattern.

### 5. Dead test guard that could never fire (test hygiene)
`test/phase_transition_test.dart` тАФ checked `c.phase == 'boon_offer'` (no
such phase; it's `'boon'`) and passed `'option'` (the command key is
`'index'`). Harmless today because that test starts runs without boons, but
a trap for the next person who flips that flag. Corrected.

## Audited тАФ judged fine / noted (no change)

- **Shop heal slot amount** is computed at stock time; buying a +max-HP
  relic first doesn't raise it. The shown number is exactly what you get тАФ
  honest, left as is.
- **Reward "RECOMMENDED"** picks by die size only (ignores tier/mods).
  Reasonable proxy; product call.
- **Ledger theme cards** buy on first tap with no confirm тАФ price is always
  visible per the ethics charter; deliberate.
- **Map walk-marker `onEnd`** doesn't fire if the screen is disposed
  mid-walk; next visit walks from one node earlier. Cosmetic, self-heals,
  guarded against missing nodes.
- **`seen_events`** keeps appending after the deck is exhausted (dup ids).
  Bounded by run length; harmless.
- **Terminal-phase autosave**: `apply` banks (queues save-file delete) then
  autosaves the terminal snapshot, so a finished run briefly leaves a
  terminal save on disk. Boot discards + deletes terminal saves, so no
  resurrection is possible; wasteful but safe.
- **Bot exact-kill hunt** ignores `attack_flat`/`elite_damage` relic hooks
  in `attackValue` (sim's assign adds them), so the bot sometimes overkills
  when it meant an exact kill. Bot-quality nit, not a sim bug; changing it
  shifts every measured baseline.
