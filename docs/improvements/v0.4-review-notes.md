# Emberdelve — improvement notes after the v0.3.2 merge (2026-07-23)

Fresh-eyes repo review by Viktor (fresh clone, Flutter 3.32.7; `flutter analyze`
clean, 77/77 tests, autoplay 200 seeds 74.0% win / 0 invalids, golden
1117081416 self-consistent). PR #1 (visual fix pass + easy/normal/hard) was
independently re-verified and merged as the v0.3.2 base. These notes are the
prioritized backlog for what comes next.

## P0 — blockers before Play closed test (all owner-gated)

1. **Real-device playthrough** (`features.json` M1-3, still the only open
   feature). Everything so far is emulator + widget-test verified; motion
   timings and the compact-combat layout were tuned blind. One session on real
   hardware (mid-range + small-screen phone) closes this.
2. **IAP full-unlock is unimplemented.** The monetization model (free +
   one-time unlock, PROJECT.md §6) has no code behind it — no `in_app_purchase`
   dependency, no gating, no restore-purchases flow. Biggest product gap;
   needs an owner decision on what the unlock gates (spec suggestion:
   characters 2–4 + ascension past 2).
3. **Play listing package**: screenshots/feature graphic, content rating
   questionnaire, privacy policy URL + data-safety form. The honest story is
   strong — zero permissions in the manifest, no network, no analytics, no
   ads — so the privacy policy is a one-pager; it just has to exist and be
   hosted somewhere.
4. **Cloud save (GPGS)**: meta progress (embers/unlocks/ascension) dies with
   the device. Undermines the long-term meta loop the whole retention design
   leans on.

## P1 — product depth & retention

5. **One boss.** Every run ends against Ember Tyrant (roster: 15 enemies =
   9 regular / 5 elite / 1 boss). 2–3 bosses with distinct mechanics (chosen
   deterministically from the map seed) would multiply run variety more per
   unit of work than any other content add.
6. **Deliberately descoped mechanics** (see docs/improvements backlogs):
   gameplay A3 chaining faces, B6 relic-combo synergies, B7 adaptive
   telegraphs; visuals #4 reward card flip, #8 launcher icon retint.
7. **Boons are thin**: 9 total, so the 1-of-3 opening offer repeats fast.
   ~15+ keeps the restart flow fresh (it's the "delve again" hook).
8. **Sprite sheets have no attack/death frames** (tween substitutes shipped in
   v0.2.0); low-HP audio danger layer also still open. Both were flagged
   "before 1.0" in earlier reviews.
9. **Balance is bot-proxied.** Bot winrate 74% normal / 91% easy / 32% hard,
   but the v0.3.1 mercy tuning already showed bot-optimal ≫ human. Closed test
   should capture a simple opt-in funnel (first-fight deaths, run length,
   difficulty distribution) — even manually-reported — before re-tuning.

## P2 — engineering health

10. **`lib/ui/screens.dart` is 2,576 lines** and every UI change lands there.
    Split into `lib/ui/screens/` per screen before it gets worse; pure
    mechanical refactor, overflow probe + widget tests are the safety net.
11. **Zero accessibility.** No `Semantics` labels anywhere; TalkBack users get
    an unlabeled custom-painted UI. Add semantics to buttons/dice/HP/intent,
    and add a `textScaleFactor: 1.3` pass to the overflow probe.
12. **No localization scaffolding** — all strings hardcoded EN. Fine for
    launch; note it before string count grows past ~300.
13. **No LICENSE file.** Public repo currently defaults to "all rights
    reserved" while PROVENANCE.md documents CC0/CC-BY *assets* — decide
    intentionally (e.g. explicit proprietary notice for code + pointer to
    PROVENANCE.md) so forks/asset reuse expectations are clear.
14. **Stale-doc gotcha**: checkpoint 04 claims fine-grained-PAT pushes never
    trigger push workflows — pushes DID trigger CI on 2026-07-23 (runs
    30044770024, 30044931004). Treat `gh workflow run` as belt-and-braces, not
    a requirement.
