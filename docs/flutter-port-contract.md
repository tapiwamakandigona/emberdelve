## 4. Porting rules (bit-parity traps)

1. **RNG call order is behavior.** Port loops so streams are consumed in the
   exact same sequence. Lua code never relies on `pairs()` order where
   randomness flows; keep every explicit ordering (sorted node ids, `ipairs`
   sequences) exactly.
2. **Integer math:** Lua uses `math.floor` division helpers; use `~/` or
   `.floor()` to match. No doubles where Lua uses integers.
3. **`#t` on arrays** = `list.length`. Lua `t[#t+1]=x` = `list.add(x)`.
4. **String iteration** in hashing is byte-wise; all sim strings stay ASCII.
5. Data values (dice mods, enemy patterns incl. the Ă—2.75 balance) copied
   **exactly** from `legacy/defold/data/*.lua` â€” no rounding, no â€ścleanupâ€ť.
6. Tests: same assertions and counts as the Lua suites (map 12, content 21,
   run 18) plus fixture-driven parity tests. Never weaken an assertion to
   make a port pass â€” a mismatch is a port bug, full stop.
